# AircraftWar
记录软件构造实验课的小项目，不断更新中，任何与代码有关的问题都欢迎大家指正~
## 第一次实验课已完成
- 采用双重锁单例模式构建英雄机
- 新建4种敌机和5种道具
- 完善了描述继承关系和英雄机单例模式的UML类图
- 精英敌机实现发射子弹和击毁后掉落道具功能
- 道具可被英雄机吸收，生效功能等待添加

## 第二次实验已完成：工厂模式重构与敌机/道具多态扩展
1. 模式重构：使用简单工厂模式 (PropFactory) 重构5种道具创建；使用工厂方法模式 (EnemyFactory) 重构4种敌机创建。
2. 架构优化：完善单例模式，将英雄机初始化参数从 Game 类中剥离，实现对象创建与使用彻底分离。
3. 多态应用：消除 Game 类中的 instanceof 判断，通过重写 dropProps() 和 active()，将道具的差异化掉落和生效逻辑下 放至各实体类。
4. 实体扩展：新增2种敌机，精锐敌机(双排弹道+横向移动)与王牌敌机(扇形弹道+横向移动)。
5. UML同步：更新工厂模式类图，确保图纸与多态重构后的代码一致。
