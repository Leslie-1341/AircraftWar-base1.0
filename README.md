# AircraftWar
记录软件构造实验课的小项目，随着课程进度不断迭代更新中。任何与代码设计或实现相关的问题，都欢迎大家指正和交流~ 

## 🚩 第一次实验：基础架构与类图设计
1. **核心架构**：采用双重检查锁定 (DCL) 的单例模式构建英雄机，确保实例唯一性。
2. **实体构建**：初步搭建游戏基础实体，新建 4 种敌机和 5 种道具。
3. **功能实现**：精英敌机实现发射子弹及击毁后掉落道具的逻辑；实现了道具被英雄机碰撞吸收的判定（具体效果待后续迭代添加）。
4. **UML 同步**：完善了系统的基础继承关系类图，绘制了英雄机的单例模式结构图。

## 🚩 第二次实验：工厂模式重构与多态扩展 
1.  **模式重构**：引入工厂模式解耦对象创建。使用**简单工厂模式 (Simple Factory)** 统一管理 5 种道具的生成；使用**工厂方法模式 (Factory Method)** 重构 4 种敌机的创建逻辑。
2.  **架构优化**：深化单例模式应用，将英雄机的初始坐标、血量等参数从 `Game` 类中剥离，实现“对象创建与使用”的分离。
3.  **多态应用**：消除 `Game` 类中的 `instanceof` 判断。通过在敌机父类定义并由子类重写 `dropProps()` 和 `active()` 方法，将“差异化掉落”与“道具生效机制”下放至各个具体实体类中。
4.  **实体扩展**：新增精锐敌机（配备双排弹道+横向移动反弹）与王牌敌机（配备扇形散射弹道+横向移动反弹）。
5.  **UML 同步**：更新了道具与敌机的工厂模式 UML 类图，使其运用多态重构后的代码严谨对齐。

## 🚩 第三次实验：策略模式重构与 Boss 敌机生成
1. **策略重构**：引入**策略模式 (Strategy Pattern)** 对弹道系统进行彻底解耦。抽象出直射、散射、环射和不发射四种具体策略，实现了英雄机在吸收特定道具后（如火力/超级火力道具）在运行时动态切换武器形态。
2. **Boss 生成**：新增 Boss 敌机及专属工厂，实现基于分数阈值的动态生成与屏幕唯一性控制。Boss 具备顶部悬浮横移、单次 20 发环形弹幕攻击的特性。
3. **机制升级**：重构掉落系统，Boss 被击毁后必定掉落 3 个道具，并实现道具按照概率差异化掉落的代码逻辑。完善了加血道具基于机体最大生命值的动态上限约束，补充了炸弹与冰冻道具的响应机制。
4. **代码瘦身与调优**：将射击属性（发射数量、威力、方向）及 `increaseHp` 恢复方法统一提取至 `AbstractAircraft` 父类，消除冗余代码；对游戏前期的射速、敌机移速及基础攻击力进行了数值平衡调优，改善玩家体验。
5. **UML 同步**：更新并绘制了弹道系统策略模式的 UML 类图，精准展示了 Context (上下文) 与各个 Strategy (策略) 之间的组合与调用关系。

## 🚩 第五次实验：Swing 界面与多线程并发
1. **GUI 架构升级 (CardLayout)**：重构了程序的启动入口，引入 `CardLayout` 管理多界面状态流转（起始菜单 -> 游戏界面 -> 排行榜），实现了完整的游戏交互闭环。
2. **MVC 表格数据绑定 (JTable)**：使用 `JTable` 与 `DefaultTableModel` 实现了排行榜数据的可视化渲染，支持榜单记录的**多行选中与安全删除**。
3. **多线程音效引擎 (MusicThread)**：重写底层音频播放机制，引入 `volatile` 线程安全标志位。实现了背景音乐（BGM）的无缝循环播放以及跨界面的“无延迟秒停”，避免了多页面切换导致的线程资源泄露。
4. **异步并发控制 (火力限时机制)**：针对火力强化道具引入了独立的后台计时线程。通过异步倒计时（`Thread.sleep`）实现了火力状态的“5秒限时生效与自动恢复”，确保主界面的事件分发线程（EDT）不被阻塞，维持游戏高帧率运行。
5. **生命周期与单例状态管理**：修复了 `HeroAircraft`（单例模式）在多轮游戏间的“状态残留”污染问题，严格规范了基于 `SwingUtilities.invokeLater` 的跨线程 UI 刷新操作。
